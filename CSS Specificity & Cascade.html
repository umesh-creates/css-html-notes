CSS Specificity & Cascade

🔥 1️⃣ Specificity क्या है?

जब एक ही element पर कई CSS rules लग रहे हों, तो कौन सा rule apply होगा — यह तय करती है Specificity।

🏆 Priority Order (सबसे ज्यादा → सबसे कम)

⬇️!important
⬇️Inline Style 
⬇️ID Selector
⬇️Class / Attribute / Pseudo-class
⬇️Element / Pseudo-element
⬇️Universal (*)

📊 Number System से समझो (Professional तरीका)

Selector Type Value
Inline Style 1000
Id 100
Class / Attribute / Pseudo-class 10
Element / Pseudo-element 1
Universal 0

🔥 Example 1 — Element vs Class

<p class="text"> Hello </p>
p {
  color: blue;   /* 1 */
}

.text {
  color: red;    /* 10 */
}

👉 Output: Red🟥
क्यों? Class (10) > Element (1)

🔥 Example 2 — Class vs ID

<p id="para" class="text">Hello</p>
.text {
  color: blue;   /* 10 */
}

#para {
  color: green;  /* 100 */
}

👉 Output: Green 🟩
क्यों? ID (100) > Class (10)

🔥 Example 2 — Class vs ID

<p id="para" class="text">Hello</p>
.text {
  color: blue;   /* 10 */
}

#para {
  color: green;  /* 100 */
}

👉 Output: Green 🟩
क्यों? ID (100) > Class (10)

🔥 Example 3 — Inline Style

<p id="para" style="color: purple;">Hello</p>
#para {
  color: red;
}

👉 Output: Purple🟪
क्यों? Inline (1000) > ID (100)

🔥 Example 4 — !important

p {
  color: blue !important;
}

#para {
  color: red;
}

👉 Output: Blue🟦
क्यों? !important सबको हरा देता है।

🔥 2️⃣ Cascade क्या है?

अगर specificity बराबर है, तो जो rule नीचे लिखा है वही apply होगा।

Example 5 — Same Selector

p {
  color: blue;
}

p {
  color: red;
}

👉 Output: Red🟥
क्यों? Specificity same (1), लेकिन red वाला rule नीचे है।

🔥 3️⃣ Equal Specificity Example

<p class="text">Hello</p>
.text {
  color: blue;
}

.text {
  color: green;
}

👉 Output: Green🟩
क्यों? Specificity same (10), तो नीचे वाला apply हुआ।

🔥 4️⃣ Combined Specificity Example

<p id="para" class="text">Hello</p>
p.text {
  color: blue;    /* 1 + 10 = 11 */
}

#para {
  color: red;     /* 100 */
}

👉 Output: Red🟥
क्यों? 100 > 11

🔥 Specificity केवल “priority list” नहीं है

Specificity actually 4 column system से calculate होती है।

Specificity Format: (a, b, c, d)

Column Meaning
a Inline style
b ID selectors
c Class / Attribute / Pseudo-class
d Element / Pseudo-element

🔥 Example 1 — Complex Selector

#main p {
  color: blue;
}

div .box p {
  color: green;
}
First selector:
  • ID = 1
  • Element = 1
  • Specificity → (0,1,0,1)
Second selector:
  • Class = 1
  • Element = 2
  • Specificity → (0,0,1,2)

अब compare करो: ID column पहले compare होगा. 1 > 0
👉 इसलिए पहला rule जीतेगा (blue)🟦

🔥 Level 2: Important Rule — Left to Right नहीं, Column wise Compare

Comparison ऐसे होता है:

1️⃣ Inline compare
2️⃣ ID compare
3️⃣ Class compare
4️⃣ Element compare 

जैसे dictionary में word compare करते हैं।

🔥 Level 3: Multiple Classes Example

.box.active {
  color: red;
}
Count:
  • Class = 2
    Specificity → (0,0,2,0)

यह single class (0,0,1,0) से ज्यादा strong है।

🔥 Level 4: Attribute और Pseudo-class भी class category में आते हैं

input[type="text"]:focus {
  border: 2px solid blue;
}
Count:
  • Element = 1 (input)
  • Attribute = 1
  • Pseudo-class = 1

Specificity: (0,0,2,1)

🔥 Level 5: Pseudo-elements कहाँ आते हैं?

p::first-letter {
  color: red;
}
  • Pseudo-element = element category

So: (0,0,0,2)

🔥 Level 6: :is() और :where():is()

  • यह अपने अंदर के selector की specificity लेता है।
:is(.box, #main) {
  color: red;
}
  • अगर #main match हुआ → specificity = ID वाली

:where()

⚠️ Specificity = 0

:where(.box) {
  color: red;
}

Specificity = (0,0,0,0)

इसलिए इसे override करना आसान है।

🔥 Level 7: Cascade Deep Rule

 Order of importance actually यह है: 

1️⃣ !important
2️⃣ Origin (User > Author > Browser default)
3️⃣ Specificity
4️⃣ Source order (नीचे वाला)
